## 版本说明

### 版本号规范
本项目使用语义化版本控制，版本号格式为：`MAJOR.MINOR.PATCH`

- **MAJOR** (主版本号): 不兼容的 API 修改
- **MINOR** (次版本号): 向下兼容的功能性新增
- **PATCH** (修订号): 向下兼容的问题修正

### 发布周期
- **主版本**: 重大功能更新或架构调整
- **次版本**: 新功能添加或重要改进
- **修订版本**: Bug 修复和小幅改进

### 版本记录规则
1. 每个版本在发布时更新此文件
2. 变更按类型分组：新增、改进、修复、废弃、移除
3. 每个变更项使用适当的 emoji 标识
4. 提供相关 issue 或 PR 链接

## 发布流程

### 1. 版本准备
```bash
# 更新版本号
echo "1.1.0" > VERSION

# 更新 changelog
# 添加新版本章节
```

### 2. 创建发布分支
```bash
git checkout -b release/v1.1.0
git add .
git commit -m "chore: 准备发布 v1.1.0"
```

### 3. 创建标签
```bash
git tag -a v1.1.0 -m "版本 1.1.0 发布"
```

### 4. 推送到远程
```bash
git push origin release/v1.1.0
git push origin v1.1.0
```

### 5. 创建 GitHub Release
1. 访问 GitHub Releases 页面
2. 点击 "Draft a new release"
3. 选择创建的标签
4. 填写发布说明
5. 上传构建产物
6. 发布

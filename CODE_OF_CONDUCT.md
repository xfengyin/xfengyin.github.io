# 贡献者公约

## 我们的承诺

为了营造一个开放、友善的环境，我们作为贡献者和维护者承诺：无论年龄、体型、身体健全与否、民族、性征、性别认同与表征、经验水平、教育程度、社会地位、国籍、相貌、种族、宗教信仰、性取向，我们项目和社区的参与者皆免于骚扰。

## 我们的准则

有助于创造积极环境的行为包括但不限于：

* **使用友好和包容性语言**
* **尊重不同的观点和经历**
* **耐心地接受建设性批评**
* **关注对社区最有利的事情**
* **与社区其他成员友善相处**

不当行为包括但不限于：

* **使用与性有关的言语或图像，以及不受欢迎的性骚扰**
* **挑衅、侮辱/贬损的评论，以及人身攻击或政治攻击**
* **公开或私下的骚扰**
* **未经许可地发布他人的个人资料，例如住址或电子邮箱**
* **其他有理由认定为违反职业操守的不当行为**

## 我们的责任

项目维护者有责任为「可接受的行为」标准做出诠释，以及对已发生的不当行为采取恰当且公平的纠正措施。

项目维护者有权利和责任去删除、编辑、拒绝违背本行为标准的评论（comments）、提交（commits）、代码、wiki 编辑、问题（issues）等贡献；项目维护者可暂时或永久地封禁任何他们认为行为不当、威胁、冒犯、有害的参与者。

## 适用范围

本行为标准适用于本项目中的所有场景，以及在本项目代表社区出现的公共场景。
代表本项目的情形包括但不限于：使用官方项目邮箱、通过官方社交媒体账号发布消息、作为指定代表参与在线或线下活动。

## 贯彻落实

可以通过 [xfengyin@example.com](mailto:xfengyin@example.com) 联系项目团队，举报滥用、骚扰或其他不当行为。

维护团队将审议并调查所有投诉，并视情况予以必要的回应。项目团队有义务对事件举报人保密。
具体执行方针或将另行发布。

未切实遵守或执行本行为标准的项目维护人员，可能会因项目负责人或其他成员的决定，被暂时或永久地剥夺参与本项目的资格。

## 举报流程

### 如何举报
如果您发现违反本行为准则的情况，请通过以下方式举报：

1. **发送邮件**: xfengyin@example.com
2. **GitHub Issues**: 创建私有 issue（如果适用）
3. **直接联系**: 通过 GitHub 私信联系维护者

### 举报内容
请提供以下信息：
- 事件发生的时间和地点
- 涉及的人员
- 事件的详细描述
- 相关证据（截图、链接等）
- 您希望的处理方式

### 处理流程
1. **确认收到**: 24小时内确认收到举报
2. **调查评估**: 1-3个工作日内完成初步调查
3. **制定方案**: 根据调查结果制定处理方案
4. **执行处理**: 执行相应的处理措施
5. **反馈结果**: 向举报人反馈处理结果（保护隐私的前提下）

## 冲突解决

### 技术争议
对于技术方案的争议，我们建议：

1. **数据驱动**: 提供性能测试数据、用户反馈等
2. **社区讨论**: 在 issue 或 discussion 中公开讨论
3. **投票决策**: 对于重大决策，可以进行社区投票
4. **维护者裁决**: 最终由维护者团队做出决定

### 人际冲突
对于人际冲突，我们建议：

1. **私下沟通**: 首先尝试私下友好沟通
2. **寻求调解**: 如果无法解决，寻求第三方调解
3. **正式投诉**: 如果冲突严重，按举报流程处理

## 社区准则

### 技术讨论
1. **尊重专业**: 尊重不同领域的技术专家
2. **理性讨论**: 基于事实和数据讨论技术问题
3. **乐于分享**: 分享知识和经验，帮助他人成长
4. **承认错误**: 勇于承认错误，及时修正

### 代码审查
1. **建设性反馈**: 提供具体、可操作的改进建议
2. **尊重劳动**: 感谢他人的贡献，即使需要修改
3. **及时响应**: 及时审查代码，提供反馈
4. **耐心指导**: 对新贡献者保持耐心，提供指导

### 内容贡献
1. **保证质量**: 确保内容准确、完整、有用
2. **尊重版权**: 遵守版权规定，注明引用来源
3. **持续改进**: 接受反馈，不断完善内容
4. **保持更新**: 及时更新过时的内容

## 维护者责任

### 行为模范
维护者应：
- 以身作则，遵守行为准则
- 积极营造友好氛围
- 及时处理不当行为
- 保护社区成员隐私

### 决策透明
维护者应：
- 公开重要决策过程
- 解释决策理由
- 听取社区反馈
- 接受合理质疑

### 公平公正
维护者应：
- 平等对待所有贡献者
- 基于事实做出判断
- 避免利益冲突
- 保持决策一致性

## 参与者权利

### 基本权利
所有参与者有权：
- 在安全、尊重的环境中参与
- 表达不同意见和观点
- 获得公平的对待和评价
- 保护个人隐私和信息安全

### 申诉权利
如果认为受到不公正对待，参与者有权：
- 了解处理决定的理由
- 提供补充信息和证据
- 请求重新评估
- 获得最终解释

## 处罚措施

### 警告
对于轻微违规行为：
- 私下或公开警告
- 要求删除不当内容
- 要求公开道歉

### 限制权限
对于较严重违规行为：
- 暂时限制 issue/PR 权限
- 暂时限制评论权限
- 暂时限制参与特定讨论

### 永久封禁
对于严重违规行为：
- 永久禁止参与项目
- 删除所有不当内容
- 报告给 GitHub 平台
